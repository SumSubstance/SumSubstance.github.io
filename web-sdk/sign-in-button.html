        # Sumsub Sign-in Button

        Simplify user onboarding and login functionality by integrating a single Sumsub Sign-in Button.

        You can choose any steps for signup as you wish (e.g. basic applicant data, questionnaire and liveness). Your customer would need to submit all necessary data once (and you can collect them, of course), and future logins will be as easy as logging in via Google or Facebook.

        Integration consists of two phases:

        • adding the button to your website
        • collect authentication information after successful sign in/up.

        # Preparations

        • Create a verification level or pick an existing one at the dashboard.
        • Note the signinId of the verification link (e.g. for https://in.sumsub.com/idensic/l/#/uni_0ZPIK8E1NI2NsmMq, it would be uni_0ZPIK8E1NI2NsmMq). It will be used upon button initialization.
        • Set up domains where the button will be hosted and a secret key for signing a JWT authentication code at the dashboard.

        # Adding the button

        # Install the Sumsub Sign-In NPM Package

        npm i @sumsub/sumsub-signin --save

        # Import the module

        import initSumsubSigninBtn from '@sumsub/sumsub-signin';

        # Add an HTML element where the button should appear:

        <a class="sns-signin-selector"></a>

        # Initialize the button

        Remember the signinId from the Preparations section.

        initSumsubSigninBtn(signinId, jwtAuthCode => {
// handler for getting a JWT auth code for passing it to the backend and verifying
  },
  {
    selector: '.sns-signin-selector' // optional, if you decide to use another class name.
  }
)

        # Verifying a JWT auth code

        Once verification is done, you will receive a JWT auth code that you must pass to your backend and verify that it's signed with the secret key as described in the "Preparation" section.

        When decoded the JWT auth code has the following shape:

        {
  "iat": 1662482016,
  "exp": 1662482616,
  "sub": "someUniqueUserId",
  "aud": "YourClientId",
  "iss": "Sumsub",
  "applicantId": "630f5ea22d7b41c750ca1f1c"
}

        # JWT properties

        Name Type Description
        iat Integer Issued At time and date (number of seconds since Unix Epoch in UTC).
        exp Integer Expires At time and date (number of seconds since Unix Epoch in UTC).
        sub String externalUserId of the applicant.
        aud String Your clientId.
        iss String Issuer.
        applicantId String Id of the applicant.
        • Verify that it's signed correctly
        • Fetch the full applicant data via API (e.g. you can get basic information via this method)
        • If everything is fine, log in your customer into your service

        We also have a demo to show you how it works after login:

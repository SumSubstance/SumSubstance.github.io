    # e-KYC

    Here, you can add the e-KYC verification step to your applicant level to pass Non-doc Verification without uploading docs and also retrieve applicant data from the official sources.

    # e-KYC API Reference

    All API requests must be authenticated as described in this section.

    # Submit applicant data

    POST /resources/applicants/{applicantId}/ekyc/submit

    # REQUEST ARGUMENTS
    Name Type Required Description
    applicantId String Yes Applicant ID.
    #{body} Object Yes An object representing applicant (see supported sources).
    # REQUEST BODY
    Name Type Required Description
    info Object Yes Applicant info.
    # info ELEMENT ATTRIBUTES
    Name Type Required Description
    country String Yes Alpha-3 country code.
    tin String Yes Tax Identifier Number
    # Example request
    Copy
    curl -X POST \
  'https://api.sumsub.com/resources/applicants/631f268442d8290001e1eee9/ekyc/submit' \
  -H 'Content-Type: application/json' \
  -d '{
        "info": {
            "country": "IND",
            "tin": "999999990019",
        }
    }'
    # RESPONSE ATTRIBUTES

    Response contains the status of the submission depends on the provider

    # RESPONSE ATTRIBUTES

    Name Type Optional Description
    status String No Status of the e-KYC flow.
    confirmationType String Yes Confirmation type.
    confirmationId String Yes Confirmation ID.
    confirmationStatus String Yes Status of the confirmation flow.
    otpConfirmation Object Yes Details for OTP confirmation if confirmationType is otp.
    # status values
    Value Description
    completed Submission is completed.
    skipped Object representing transaction review result.
    confirmationRequired Submission needs approval. See confirmationType.
    rejected Submission was rejected.

    Access to applicant's data from some providers need to be confirmed by applicant. In that case status will be set to confirmationRequired and confirmationStatus to created. Confirmation type will be provided in confirmationType field. How to confirm submission see below

    E.g. For an Indian aadhaar, applicant receives the SMS with OTP code on the phone number that is linked to that aadhaar. Then the applicant provides the code to confirm access.

    # confirmationType values
    Value Description
    otp Submission should be confirmed via OTP code.
    oAuth Applicant should be directed to external service to proceed submission.
    # otpConfirmation ELEMENT ATTRIBUTES
    Name Type Description
    codeLength Number OTP code length
    # oAuthConfirmation ELEMENT ATTRIBUTES
    Name Type Description
    url String External auth URL where applicant should be directed to pass authentication.
    redirectUrl String Final URL address where auth page will be redirected after applicant finish the journey.

    Applicant should be redirected to the url where they can authenticate the external source. At the end of the auth journey, applicant will be redirected to the completeUrl consisting of redirectUrl enriched with queryParams with the auth result.

    Several redirects could be performed during the auth flow. To detect the completion of the journey track, redirect to the redirectUrl.
    To complete ekyc oAuth flow, pass final full url as completeUrl to the [confirm request] ( /api-reference/ekyc.html#confirm-applicant-data)

    # Example response
    Copy
    //no confirmationType
{
    "status": "completed",
}
//confirmationType: otp
{
    "status": "confirmationRequired",
    "confirmationId": "bzaPxnCpyxEqs0sJKASXxZIk",
    "confirmationStatus": "created",
    "confirmationType": "otp",
    "otpConfirmation": {
        "codeLength": "6",
    }
}
//confirmationType: oAuth
{
  ...
  "confirmationType" : "oAuth",
  "oauthConfirmation" : {
    "url" : "https://ext.auth.service.com/oidc/authorize?param1=value1&param2=value2",
    "redirectUrl" : "https://in.sumsub.com/idensic/ekyc/redirect"
  }
}

    # Confirm applicant data

    POST /resources/applicants/{applicantId}/ekyc/confirm/{confirmationid}

    # REQUEST ARGUMENTS
    Name Type Required Description
    applicantId String Yes Applicant ID.
    confirmationId String Yes confirmationId from submission response.
    #{body} Object Yes An object representing confirmation data (see example).
    # REQUEST BODY
    Name Type Required Description
    otp Object No OTP confirmation data if confirmationType is otp.
    oauth Object No oAuth confirmation data if confirmationType is oAuth.
    # otp ELEMENT ATTRIBUTES
    Name Type Required Description
    code String Yes OTP that was sent to applicant.
    # Example request
    Copy
    curl -X POST \
  'https://api.sumsub.com/resources/applicants/631f268442d8290001e1eee9/ekyc/confirm/bzaPxnCpyxEqs0sJKASXxZIk' \
  -H 'Content-Type: application/json' \
  -d '{
        "otp": {
            "code": "123456",
        }
    }'
    # oauth ELEMENT ATTRIBUTES
    Name Type Description
    completeUrl String Final redirected URL of the auth journey with query params. Format: redirectUrl?<queryParams>.
    # Example request
    Copy
    curl -X POST \
  'https://api.sumsub.com/resources/applicants/631f268442d8290001e1eee9/ekyc/confirm/bzaPxnCpyxEqs0sJKASXxZIk' \
  -H 'Content-Type: application/json' \
  -d '{
        "oauth": {
            "completeUrl": "https://in.sumsub.com/idensic/ekyc/redirect?code=rUKrpIoRZ44ENZnz8EC6FP",
        }
    }'
    # RESPONSE ATTRIBUTES

    Response contains the status of the confirmation

    # RESPONSE ATTRIBUTES

    Name Type Optional Description
    status String No Status of the e-KYC flow.
    confirmationType String Yes Confirmation type.
    confirmationId String Yes Confirmation ID.
    confirmationStatus String Yes Status of the confirmation flow.
    otpConfirmation Object Yes Details for OTP confirmation if confirmationType is otp.
    # confirmationStatus values

    Status of the confirmation for that confirmationId E.g For otp confirmation. Right code - confirmed, wrong code for several times - retry, limit of retries was exceeded - rejected

    Value Description
    created Confirmation was created.
    confirmed Confirmation was confirmed.
    retry Confirmation with confirmationId was temporary rejected and could be retried.
    rejected Confirmation with confirmationId was rejected. Start from submission again or skip the e-KYC step.
    # Example response
    Copy
    {
    "status": "confirmationRequired",
    "confirmationType": "otp",
    "confirmationId": "bzaPxnCpyxEqs0sJKASXxZIk",
    "confirmationStatus": "retry",
    "otpConfirmation": {
        "codeLength": "6",
    }
}

    # Error codes

    Also. please pay attention to the failed requests. More common error descriptions can be found here.

    Code Description
    9100 Invalid data provided. Check correctness and try again.
    9101 You cannot be verified with this method right now.
    9102 The source that we rely on is not available right now.
    9103 The source is experiencing difficulties right now.
    9104 No sufficient data in the source.
    9105 Max number of attempts exceeded.
    9107 Violations of mismatches found

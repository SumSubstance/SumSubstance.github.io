      # Enhanced Proof of Address document validation

      # Introduction

      Our service has an additional feature that can identify the type of issuing company for the proof of address you provide us with, as well as its subtype.

      For the feature to be available, please inform the Sum&Substance team, because certain settings have to be set up on our side.

      # PoA Data API

      After sending us the required documents, initiating check and waiting for a result (following steps described here or using our SDKs), you will be able to fetch results from our side. They will contain data recognized from provided documents.

      # Getting main PoA data

      GET /resources/applicants/{applicantId}/one

      or

      GET /resources/applicants/-;externalUserId={externalUserId}/one

      # REQUEST ARGUMENTS
      Name Type Required Description
      applicantId String Yes Applicant ID.
      externalUserId String Yes User id in your system.
      # RESPONSE
      Response Code Response Reason
      200 Applicant entity
      400 Error JSON Invalid applicant ID provided.
      404 Error JSON Applicant not found.

      It's basically an applicant that you've created (or we've created for you, e.g., via WebSDK) with augmented information in cases where a verification has been completed. The info.idDocs[] array of objects contains information extracted from applicants’ documents.

      Descriptions below will contain only data about Proof of Address (idDocType: "UTILITY_BILL") document type and its fields. You can find more info about other documents here.

      # ATTRIBUTES FOR PoA idDocs ARRAY ELEMENTS
      Name Type Optional Description
      idDocType String No Document type identifier. Has UTILITY_BILL/UTILITY_BILL2 for all PoA documents.
      country String No 3-letter Alpha-3 country code.
      firstName String Yes Customer's given or full name depending on the possibility to separate names at the document.
      firstNameEn String Yes Customer's given or full name transliteration.
      lastName String Yes Customer's last name.
      lastNameEn String Yes Customer's last name transliteration.
      middleName String Yes Customer's middle name.
      middleNameEn String Yes Customer's middle name transliteration.
      issuedDate String Yes Date of PoA issue (format YYYY-mm-dd, e.g. 2021-01-26).
      address Object Yes Array of customer's address fields.
      # address ELEMENTS FIELDS
      Name Type Description
      country String Alpha-3 country code.
      postCode String Postal code.
      town String Town or city name.
      townEn String Transliteration of own or city name.
      street String Street name or full address depending on the provided document format.
      streetEn String Transliteration of street name or full address depending on the provided document format.
      subStreet String Additional street information.
      subStreetEn String Transliteration of additional street information.
      state String State/province name if applicable.

      # Getting additional PoA data

      GET /resources/checks/latest?type=POA&applicantId={applicantId}

      # REQUEST ARGUMENTS
      Name Type Required Description
      applicantId String Yes Applicant ID.
      # RESPONSE

      Contains a list of PoA checks: returns array of objects.

      Name Type Optional Description Available values
      answer String No Answer on overall PoA check. GREEN/YELLOW/RED
      subType String No PoA subtype. Described below
      companyContact Object Yes PoA issuer type (.type). Described below
      # AVAILABLE VALUES FOR .subType AND companyContact.type ATTRIBUTES
      companyContact.type subType
      governmentOrganization statement/voterRegistration/taxBill/other
      utilityProvider telecom/utilityBill/other
      bank bankStatement/bankLetter/other
      mobileOperator
      other lease/other
      # Example request
      Copy
      curl -X GET \
  'https://api.sumsub.com/resources/checks/latest?type=POA&applicantId=6b594ade0a975a36c9349e66'

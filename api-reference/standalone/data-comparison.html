# Standalone data comparison

All API requests must be authenticated as described in this section.

# Name cross-validation

This method takes a pair of document data from request body and compares names within them.

POST /resources/checks/crossCheck?comparisonMode={comparisonMode}

# REQUEST ARGUMENTS
Name Type Required Description
comparisonMode String Yes Data comparison mode.
#{body} List of Objects Yes .idDocs. List with two objects representing documents with data.

List within .idDocs can take only two objects.

# Available values of comparisonMode
Name Description
strict Allows only an exact match: Dr. John Smith and John Smith are acceptable, John D Smith and John Smith are not.
containment Compares the most similar name parts separately: Emanuel Burns and Burns Emanuel are acceptable, but Mike Wazowski and Michael Wazowski will not be acceptable.
fuzzyContainment Allows greater flexibility in spelling and accept rearrangement of given and family names: Judy Smith, Smith Judith, J Smith, and Smith J M are acceptable.
def Allows a wide range of spelling: John M Smith and Dr. J Manuel Smith are acceptable.
# .idDocs ELEMENT ATTRIBUTES
Name Type Required Description
firstName String Yes First name
middleName String No Middle name
lastName String No Last name
# RESPONSE

JSON representing check result.

Name Type Description
fullNameMatch Boolean Full name comparison result.
# Example request
Copy
curl -X POST \
  'https://api.sumsub.com/resources/checks/crossCheck?comparisonMode=fuzzyContainment' \
  -H 'Content-Type: application/json' \
  -d '{
          "idDocs": [
              {
                  "idDocType": "UTILITY_BILL",
                  "country": "USA",
                  "firstName": "Judy Smith"
              },
              {
                  "idDocType": "PASSPORT",
                  "country": "USA",
                  "firstName": "Judith",
                  "lastName": "Smith"
              }
          ]
    }
# Example response
Copy
{
  "fullNameMatch": true
}

# Standalone identifier confirmation via code

All API requests must be authenticated as described in this section.

# Submission

This method will send an SMS or email with a code to the provided identifier (phone number or email address) so it could be verified later.

POST /resources/identifierConfirmations/-/request

# REQUEST BODY
Name Type Required Description
identifier String Yes Phone number (E.164 format) or email address
identifierType String Yes Type of identifier, e.g. phone or email
targetId String Yes External client ID involved in the rate limiter
# RESPONSE
Name Type Description
id String Verification request ID
createdAt Date Verification request creation date
identifier String Phone number or email
verificationCodeLength Integer Length of verification code
identifierType String Type of identifier, e.g. phone or email
status String Verification request status (created/verified/retry/rejected)
targetType String Target type
# Example request
Copy
curl -X POST \
  'https://api.sumsub.com/resources/identifierConfirmations/-/request' \
  -H 'Content-Type: application/json' \
  -d '{
        "identifier": "+35799669637",
        "identifierType": "phone"
      }'
# Example response
Copy
{
    "id": "IN5f8wKlaTdP8NzHmmscbGhz",
    "createdAt": "2023-05-12 14:04:21",
    "identifier": "+35799669637",
    "verificationCodeLength": 6,
    "identifierType": "phone",
    "targetType": "standalone",
    "status": "created"
}

# Confirmation

Confirmation of the code that was sent via SMS by method above.

POST /resources/identifierConfirmations/{verificationId}/verify?code={code}

# REQUEST ARGUMENTS
Name Type Required Description
verificationId String Yes Verification request ID that should be received via submission
code String Yes Verification code that was sent via SMS or email on submission request
# RESPONSE
Name Type Description
id String Verification request ID
createdAt Date Verification creation date
identifier String Phone number or email address
verificationCodeLength Integer Length of verification code
identifierType String Type of identifier (email/phone)
status String Verification request status (created/verified/retry/rejected)
# Example request
Copy
curl -X POST \
  'https://api.sumsub.com/resources/identifierConfirmations/IN5f8wKlaTdP8NzHmmscbGhz/verify?code=517972'
# Example response
Copy
{
    "id": "IN5f8wKlaTdP8NzHmmscbGhz",
    "createdAt": "2023-05-12 14:04:21",
    "identifier": "+35799669637",
    "verificationCodeLength": 6,
    "identifierType": "phone",
    "status": "verified"
}

# Rate limits

We care about preventing spam, therefore we have limits:

Field RPS
identifier 1 request / 1 min
targetId 1 request / 1 min
targetId 10 requests / 1 hour
licenseKey (only for sandbox) 10 requests / 24 hours

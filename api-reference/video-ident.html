      # Video Ident API

      Video Ident API methods allows you to get all required data gathered during video call. Please check for Video Ident product at our help center.

      All API requests must be authenticated as described in this section.

      # Getting video call data

      With the method below, you'll be able to get video call results and fetch call identifier to upload video later.

      GET /resources/inspections/{inspectionId}?fields=videoIdentData

      # REQUEST ARGUMENTS
      Name Type Required Description
      inspectionId String Yes Inspection ID
      # Example request
      Copy
      curl -X GET \
  'https://api.sumsub.com/resources/inspections/5e9412223cc1813b4db0b0e4?fields=videoIdentData'
      # RESPONSE ATTRIBUTES
      Name Type Optional Description
      id String No Inspection ID
      videoIdentData Object Yes Object representing data about video calls
      # videoIdentData ATTRIBUTES
      Name Type Optional Description
      reviewStatus String No Current status of Video Ident step. Options for reviewStatus can be found here.
      moderatorDisplayName String Yes Video Call operator name.
      compositions Array of objects Yes Data about each video call.
      # compositions ELEMENTS ATTRIBUTES
      Name Type Optional Description
      compositionCreatedAt Date No Date and time of video call start.
      compositionDuration Integer Yes Video call duration in seconds.
      compositionMediaId String Yes Video Call record identifier.
      # Example response
      Copy
      {
    "id": "5e9412223cc1813b4db0b0e4",
    "videoIdentData": {
        "reviewStatus": "completed",
        "moderatorDisplayName": "Bradley Peak",
        "compositions": [
            {
                "compositionCreatedAt": "2022-07-26 09:02:53",
                "compositionDuration": 1868,
                "compositionMediaId": "62dfae546cef4a0001079b37"
            },
            {
                "compositionCreatedAt": "2022-07-27 07:53:31",
                "compositionDuration": 1868,
                "compositionMediaId": "62e0ef8d7ff76f000176aeb1"
            },
            {
                "compositionCreatedAt": "2022-07-29 07:21:19",
                "compositionDuration": 540,
                "compositionMediaId": "62e38aa5fd6916000172dc23"
            }
        ]
    }
}

      # Getting video call media (video)

      GET /resources/videoIdent/applicant/{applicantId}/media/{compositionMediaId}

      # REQUEST ARGUMENTS
      Name Type Optional Description
      applicantId String No Applicant ID
      compositionMediaId String No Video call media identifier
      # Example request
      Copy
      curl -X GET \
  'https://api.sumsub.com/resources/videoIdent/applicant/5e9412223cc1813b4db0b0e3/media/{compositionMediaId}'
      # RESPONSE

      Binary content representing a video call recording. The Content-Type response header precisely describes the response mime-type. The full list of mime-types and file formats you can find here.
